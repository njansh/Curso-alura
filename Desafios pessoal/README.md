\# Desafios Pessoais — Java Back-end



Esta pasta reúne desafios autorais em Java, desenvolvidos de forma independente com o objetivo de consolidar e aprofundar conhecimentos em desenvolvimento back-end, indo além dos exercícios guiados por cursos.



Os projetos aqui presentes foram pensados para simular cenários mais próximos da realidade, exigindo maior capacidade de modelagem, organização de código, aplicação de regras de negócio e tomada de decisões técnicas.



\## Objetivo desta Pasta



\- Praticar Java puro (Core Java) de forma estruturada

\- Aplicar conceitos de orientação a objetos em cenários mais complexos

\- Desenvolver soluções sem roteiro pronto, estimulando autonomia

\- Criar projetos com valor de portfólio, voltados para leitura por recrutadores e outros desenvolvedores



\## O que você encontrará aqui



\- Sistemas acadêmicos, administrativos ou simuladores de negócio

\- Uso de:

&nbsp; - Encapsulamento

&nbsp; - Herança

&nbsp; - Polimorfismo

&nbsp; - Enums

&nbsp; - Regras de validação

&nbsp; - Organização em pacotes (modelo, principal, etc.)

\- Código escrito com foco em:

&nbsp; - Clareza

&nbsp; - Coesão

&nbsp; - Manutenibilidade



Cada desafio possui sua própria estrutura e, quando necessário, um README específico explicando o contexto e as regras do sistema.



\## Organização Geral



&nbsp;   desafios\_pessoais/

&nbsp;   ├── desafio1/

&nbsp;   ├── desafio2/

&nbsp;   └── ...



Cada pasta de desafio contém um projeto Java independente, normalmente seguindo o padrão:



\- modelo/ → classes de domínio

\- principal/ → classe com método main para execução e testes



\## Observação Importante



Esta pasta é independente dos projetos desenvolvidos durante a Formação Praticando Java (Alura).  

Ela representa um passo adicional na evolução técnica, focado em desafios próprios e aprendizado ativo.
